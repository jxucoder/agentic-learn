What this solution does (different from prior HistGradientBoosting approach):
- Uses `LogisticRegression` on engineered sparse features instead of gradient boosting trees.
- Handles missing values via imputers plus explicit missingness indicators.
- Engineers non-linear and interaction terms (`log_*`, squared terms, `income_x_hours`, `distance_x_hours`, `age_x_income`, `region_education`, `age_bucket`).
- Handles categorical features with `OneHotEncoder`.
- Identifies/handles likely noise features using `SelectPercentile(mutual_info_classif, percentile=65)` inside the CV pipeline.
